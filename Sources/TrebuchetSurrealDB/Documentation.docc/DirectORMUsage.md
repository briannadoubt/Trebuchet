# Direct ORM Usage

Build distributed actors with direct SurrealDB ORM access for full control over your data models.

## Overview

The direct ORM pattern gives you complete control over your data models while still benefiting from type-safe queries and automatic schema generation.

## Defining Models

Create models using Swift structs with the `Codable` and `Sendable` protocols:

```swift
import SurrealDB

struct Todo: Codable, Sendable {
    @ID var id: String?
    var text: String
    var completed: Bool
    var actorId: String
    var createdAt: Date

    static var tableName: String { "todos" }
}
```

### Property Wrappers

Use the `@ID` property wrapper for automatic record ID handling:

```swift
@ID var id: String?  // Auto-generated by SurrealDB
@ID(strategy: .uuid) var id: RecordID?  // UUID generation
@ID(strategy: .ulid) var id: RecordID?  // ULID generation
```

## Schema Generation

Generate database schema from your Swift types:

```swift
@Trebuchet
distributed actor TodoList {
    private let db: SurrealDB

    init(actorSystem: TrebuchetActorSystem, db: SurrealDB) async throws {
        self.actorSystem = actorSystem
        self.db = db

        // Auto-generate schema
        try await db.defineTable(for: Todo.self)
    }
}
```

This generates and executes:
```sql
DEFINE TABLE todos SCHEMAFULL;
DEFINE FIELD text ON todos TYPE string;
DEFINE FIELD completed ON todos TYPE bool;
DEFINE FIELD actorId ON todos TYPE string;
DEFINE FIELD createdAt ON todos TYPE datetime;
```

## CRUD Operations

### Create

```swift
distributed func addTodo(_ text: String) async throws -> Todo {
    var todo = Todo(
        text: text,
        completed: false,
        actorId: id.id,
        createdAt: Date()
    )
    return try await db.create(todo)
}
```

### Read

```swift
// Get all todos for this actor
distributed func getTodos() async throws -> [Todo] {
    return try await db.query(
        Todo.self,
        where: [\.actorId == id.id]
    )
}

// Get a specific todo by ID
distributed func getTodo(_ todoId: String) async throws -> Todo? {
    return try await db.select(todoId) as Todo?
}
```

### Update

```swift
distributed func toggleTodo(_ todoId: String) async throws {
    guard var todo = try await db.select(todoId) as Todo? else {
        throw TodoError.notFound
    }
    todo.completed.toggle()
    _ = try await db.update(todo)
}

distributed func updateTodoText(_ todoId: String, text: String) async throws {
    guard var todo = try await db.select(todoId) as Todo? else {
        throw TodoError.notFound
    }
    todo.text = text
    _ = try await db.update(todo)
}
```

### Delete

```swift
distributed func deleteTodo(_ todoId: String) async throws {
    try await db.delete(todoId)
}

distributed func clearCompleted() async throws {
    let completed = try await db.query(
        Todo.self,
        where: [
            \.actorId == id.id,
            \.completed == true
        ]
    )
    for todo in completed {
        guard let id = todo.id else { continue }
        try await db.delete(id)
    }
}
```

## Type-Safe Queries

Use KeyPath-based queries for type safety:

### Simple Conditions

```swift
// Single condition
let incompleteTodos = try await db.query(
    Todo.self,
    where: [\.completed == false]
)
```

### Multiple Conditions

```swift
// Multiple conditions (AND logic)
let recentIncompleteTodos = try await db.query(
    Todo.self,
    where: [
        \.completed == false,
        \.createdAt > Date().addingTimeInterval(-86400)  // Last 24 hours
    ]
)
```

### Ordering

```swift
// Order by creation date, newest first
let todosByDate = try await db.query(
    Todo.self,
    where: [\.actorId == id.id],
    orderBy: [(\.createdAt, false)]  // false = descending
)

// Order by multiple fields
let sortedTodos = try await db.query(
    Todo.self,
    orderBy: [
        (\.completed, true),   // Complete last
        (\.createdAt, false)   // Newest first within each group
    ]
)
```

### Limiting Results

```swift
// Get the 10 most recent todos
let recentTodos = try await db.query(
    Todo.self,
    where: [\.actorId == id.id],
    orderBy: [(\.createdAt, false)],
    limit: 10
)
```

## Aggregations

Compute statistics over your data:

```swift
struct TodoStats: Codable, Sendable {
    let total: Int
    let completed: Int
    let incomplete: Int
}

distributed func getStats() async throws -> TodoStats {
    let todos = try await db.query(
        Todo.self,
        where: [\.actorId == id.id]
    )

    return TodoStats(
        total: todos.count,
        completed: todos.filter(\.completed).count,
        incomplete: todos.filter { !$0.completed }.count
    )
}
```

## Actor Isolation

Ensure data is isolated to each actor instance:

```swift
// All queries should filter by actorId
distributed func getTodos() async throws -> [Todo] {
    return try await db.query(
        Todo.self,
        where: [\.actorId == id.id]  // ← Important for isolation
    )
}

// Verify ownership before updates
distributed func updateTodo(_ todoId: String, text: String) async throws {
    guard var todo = try await db.select(todoId) as Todo? else {
        throw TodoError.notFound
    }

    // Verify this actor owns the todo
    guard todo.actorId == id.id else {
        throw TodoError.unauthorized
    }

    todo.text = text
    _ = try await db.update(todo)
}
```

## Error Handling

Handle SurrealDB-specific errors:

```swift
enum TodoError: Error {
    case notFound
    case unauthorized
    case invalidData
}

distributed func deleteTodo(_ todoId: String) async throws {
    do {
        try await db.delete(todoId)
    } catch let error as SurrealError {
        switch error {
        case .invalidRecordID:
            throw TodoError.notFound
        default:
            throw error
        }
    }
}
```

## Best Practices

1. **Always filter by actorId** to ensure data isolation
2. **Verify ownership** before updates or deletes
3. **Use type-safe queries** instead of raw SurrealQL
4. **Generate schema in init** for automatic table creation
5. **Handle errors gracefully** with custom error types
6. **Use aggregations** instead of loading all records
