# Trebuchet Streaming Debug Prompt

## Context

I'm building a distributed actor framework called Trebuchet that enables real-time state streaming from server-side actors to SwiftUI clients. The framework uses WebSocket for transport and Swift's distributed actors for RPC.

## Architecture

**Server Side:**
- `TrebuchetServer` hosts distributed actors
- Detects streaming methods (those starting with "observe" and returning `AsyncStream`)
- Has a `streamingHandler` that should return `AsyncStream<Data>` for streaming methods
- Iterates the stream and sends `StreamData` envelopes over WebSocket

**Client Side:**
- `@ObservedActor` property wrapper in SwiftUI
- Calls `remoteCallStream()` to subscribe to actor state changes
- Should receive `StreamStart`, `StreamData`, and `StreamEnd` envelopes

**Streaming State Pattern:**
```swift
@Trebuchet
public distributed actor ConversationActor {
    @StreamedState public var messages: [Message] = []

    // Generated by @Trebuchet macro:
    // nonisolated public func observeMessages() -> AsyncStream<[Message]>
}
```

## The Problem

We're trying to generate `nonisolated` observe methods that can be called from the streaming handler (which runs outside the actor's isolation domain). The methods need to:

1. Be callable from outside the actor (hence `nonisolated`)
2. Access actor-isolated properties like `_messages_storage` and `_messages_continuations`
3. Return an `AsyncStream` that yields the actor's state

## Current Errors

### Error 1: Actor Isolation
```
error: distributed actor-isolated property '_messages_continuations' can not be accessed from a nonisolated context
```

**Generated Code (Xcode projects):**
```swift
nonisolated public func observeMessages() -> AsyncStream<[Message]> {
    let id = UUID()
    let actor = self // Capture actor reference

    return AsyncStream { continuation in
        Task.detached {
            await actor._messages_continuations[id] = nil
            await actor._messages_continuations[id] = continuation
            continuation.yield(await actor._messages_storage)
        }

        continuation.onTermination = { @Sendable [weak actor, id] _ in
            Task {
                try? await actor?._cleanupMessagesContinuation(id)
            }
        }
    }
}
```

### Error 2: Calling from Streaming Handler
The server's streaming handler receives `(InvocationEnvelope, any DistributedActor)` and needs to call the observe method:

```swift
await server.configureStreaming { envelope, actor in
    guard let typedActor = actor as? ConversationActor else {
        throw TrebuchetError.remoteInvocationFailed("Invalid actor type")
    }

    switch envelope.targetIdentifier {
    case "observeMessages":
        let stream = typedActor.observeMessages()  // How to call this?
        return TrebuchetServer.encodeStream(stream)
    default:
        throw TrebuchetError.remoteInvocationFailed("Unknown method")
    }
}
```

## Constraints

1. **AsyncStream is not Codable** - Can't make observe methods `distributed` (distributed methods require Codable return types)
2. **Actor Isolation** - Can't call non-distributed methods on distributed actors from outside their isolation
3. **Single WebSocket Connection** - We want both RPC and streaming over one connection (not separate channels)
4. **Type Safety** - Should maintain compile-time type safety

## What We've Tried

### Attempt 1: Distributed Methods ❌
```swift
public distributed func observeMessages() -> AsyncStream<[Message]>
```
**Failed:** AsyncStream is not Codable

### Attempt 2: Regular Actor Methods ❌
```swift
public func observeMessages() async -> AsyncStream<[Message]>
```
**Failed:** Can't call from outside actor isolation ("only 'distributed' instance methods can be called")

### Attempt 3: Nonisolated with Task ❌
```swift
nonisolated public func observeMessages() -> AsyncStream<[Message]> {
    return AsyncStream { continuation in
        Task {
            await _messages_continuations[id] = continuation
        }
    }
}
```
**Failed:** Regular `Task` in nonisolated context can't access actor properties

### Attempt 4: Nonisolated with Task.detached ❌ (Current)
```swift
nonisolated public func observeMessages() -> AsyncStream<[Message]> {
    let actor = self
    return AsyncStream { continuation in
        Task.detached {
            await actor._messages_continuations[id] = continuation
        }
    }
}
```
**Failing:** Still getting isolation errors

## Questions

1. **How can a nonisolated method on a distributed actor access actor-isolated properties?**
   - Is `Task.detached` the right approach?
   - Do we need `@unchecked Sendable` or `unsafeBitCast`?

2. **Is there a better pattern for streaming from distributed actors?**
   - Should we use reverse callbacks (server calls client actor)?
   - Should we use separate WebSocket channels for data streaming?
   - Should we abandon nonisolated and use a different approach?

3. **How do other Swift distributed actor frameworks handle streaming?**
   - Does Swift Distributed Actors have a pattern for this?
   - Are we fighting the type system too hard?

## Ideal Solution

The ideal solution would:
- ✅ Allow server to call observe methods on local actors
- ✅ Work with Swift's distributed actor isolation model
- ✅ Maintain type safety
- ✅ Use single WebSocket connection
- ✅ Support both manual macro expansion (Xcode) and compiler macros (SPM)
